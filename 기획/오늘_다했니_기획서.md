# 오늘 다 했니? — 서비스 기획서

> 작성 기준: 2026-05-11 / 리포지터리 문서 및 코드 분석 기반

---

## 1. 프로젝트 개요

### 1.1 서비스 소개

**오늘 다 했니?** 는 하루 생활계획표를 세우고, 실천을 유도하며, 완료 여부를 기록하게 해주는 **습관 형성 지원 앱**이다.

계획만 하고 실천하지 못하는 문제를 해결하기 위해, 시간 단위의 세밀한 계획 설정과 실시간 리마인드, 미수행 계획 회복 흐름을 하나의 루프로 연결한다.

### 1.2 기획 배경 및 목적

| 항목 | 내용 |
|------|------|
| 배경 | 계획만 하는 것이 아닌, **실천까지 관리**하기 위해 |
| 목적 | 습관을 만들기 위한 세밀한 계획과 **계획 실천 시스템** 구축 |
| 핵심 가치 | 계획 인지 → 완료 행동 → 회복 흐름의 단일 루프 |

### 1.3 타깃 사용자

- 일정을 계획하고 실천하는 사람
- 해야 할 일을 계획했지만 잊거나 미루는 사람
- 하루 단위의 루틴을 지속적으로 유지하고 싶은 사람

## 2. 플랫폼 전략

### 2.1 단계별 플랫폼 전략

```
1단계: 웹 MVP (Next.js)
  └─ 핵심 UX 검증, 도메인 로직 고정

2단계: 앱 전환 (Expo + React Native)  ← 현재 진행 중
  └─ 실제 앱 경험, 로컬 알림, 네이티브 UI

3단계: 앱 안정화 및 출시
  └─ 앱 스토어 배포, 운영 루프
```

### 2.2 웹 MVP의 역할

웹은 최종 제품이 아니라 앱 전환 전 검증 단계다.

**웹에서 검증할 것:**
- 원형 24시간 플래너가 "지금 해야 할 계획" 인지에 유리한가
- 시간 입력/수정 흐름이 첫 사용자에게 과도한 부담인가
- 현재 계획 강조 → 완료 처리 흐름이 한 세션에서 이어지는가
- 리마인드가 실천 전환에 실제 의미가 있는가

### 2.3 앱 전환 기준 (앱 전환 실행 준비 단계 도달)

| 조건 | 현재 상태 |
|------|-----------|
| 핵심 사용자 여정 테스트로 고정 | 완료 |
| 도메인 로직/provider 경계 분리 | 완료 |
| 저장소 계약 추상화 | 완료 |
| 리마인드 기본 정책 고정 | 완료 |
| 회복 흐름 UX 검증 | 완료 |

## 3. 핵심 제품 철학

### 3.1 핵심 사용자 루프

```
[오늘 화면 진입]
  ↓
[현재 계획 인지]  ← 24시간 원형 플래너 + 현재 시각 포인터
  ↓
[리마인드 수신]   ← 시작 5분 전 ~ 시작 후 10분 창
  ↓
[완료 처리]       ← 완료 토글
  ↓ (미완료 시)
[missed 전환]     ← 종료 시각 경과 후 자동
  ↓
[회고 또는 다시 지정]  ← 회복 루프
  ↓
[월간 동기부여 확인]   ← 지속 동기 신호
```

### 3.2 제품 설계 원칙

- **실행 흐름 우선**: 오늘 화면은 실행 중심이며, 분석/통계 허브가 아니다
- **단순한 회복**: 놓친 계획도 포기하지 않고 다시 이어갈 수 있는 최소 경로 제공
- **알림 최소화**: 과한 알림보다 적시 1회 알림으로 집중
- **점수가 아닌 흐름**: 경쟁/랭킹이 아니라 지속 동기 신호 중심

## 4. 핵심 기능 명세

### 4.1 계획 관리

#### 계획 데이터 모델

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 계획 고유 ID |
| `date` | `YYYY-MM-DD` | 계획 날짜 (월간 집계 기준) |
| `title` | string | 계획 제목 |
| `startMinute` | number | 시작 시각 (분 단위) |
| `endMinute` | number | 종료 시각 (분 단위) |
| `status` | `PlanStatus` | 상태 |
| `color` | string | 색상 |
| `rescheduleCount` | number | 다시 지정 횟수 (최대 3회) |
| `sourcePlanId` | string? | 원본 계획 ID (재지정 체인 추적) |
| `reflectionNote` | string? | 회고 메모 |

#### 계획 상태 전이

```
pending
  ├─ [완료 토글] ──→ done
  └─ [종료 시각 경과] ──→ missed
       ├─ [회고 저장] ──→ (회고 완료)
       └─ [다시 지정] ──→ pending (새 일정, 최대 3회)
```

#### 핵심 규칙

- **시간 겹침 차단**: 기존 계획과 겹치는 시간 입력 불가
- **다시 지정 최대 3회**: `rescheduleCount`로 추적
- **자동 축약 없음**: 빈 시간이 부족하면 다시 지정 실패, 사용자가 직접 짧게 조정
- **시작 전 완료 버튼 비활성**: 시작 시각 전에는 완료 불가

### 4.2 리마인드 시스템

#### 리마인드 종류

| 종류 | OS 알림 | 인앱 UI | 시점 |
|------|---------|---------|------|
| 시작 리마인드 | O | O (배너) | 시작 5분 전 ~ 시작 후 10분 |
| 종료 5분 전 | O | O (배너) | 종료 5분 전 |
| 회복 알림 (missed) | X | O (카드/시트) | 종료 후 |

#### 시작 리마인드 규칙

- 창: 시작 5분 전 ~ 시작 후 10분, **1회만 표시**
- 시작 전: `닫기`만 노출
- 시작 후: `지금 완료` 또는 `닫기`

#### 종료 5분 전 규칙

- 액션: `계속 진행` **1개만** (회고/다시 지정은 종료 직후 카드에서 처리)
- 이유: 종료 전에 추가 행동 유도는 부담 → 확인 용도만 유지

#### 제외 기능 (1차 버전)

- 스누즈
- 반복 알림
- 다단계 리마인드
- 알림 우선순위 세분화

### 4.3 missed 회복 흐름

#### 회복 재노출 규칙

| 상태 | 조건 | 표시 내용 |
|------|------|-----------|
| 회고 다시 보기 | 회고 메모 없음 | 회고 입력 유도 |
| 다시 지정 다시 보기 | 회고는 있으나 pending 후속 없음, 재지정 가능 | 다시 지정 유도 |
| 다시 지정됨 | pending 후속이 60분보다 멀리 있음 | 예약 상태 표시 |
| 다시 지정 곧 시작 | pending 후속이 60분 이내 | 재강조 |
| 회복 진행 중 | pending 후속이 현재 진행 중 | 재강조 |

#### 회복 정책 조정 기준

아래 조건이 반복되면 정책 조정 검토:
- `회고 다시 보기` 3회 이상, 실제 회고 저장 미발생
- `다시 지정 곧 시작` 3회 이상, "너무 이르다" 반응
- `다시 지정 불가` 2회 이상, 사용자가 포기/삭제 선택

### 4.4 월간 동기부여

#### 제공 지표

| 지표 | 설명 |
|------|------|
| 이번 달 완료 개수 | `done` 상태 계획 수 |
| 이번 달 완료율 | `done / 전체 계획` |
| 이번 달 missed 개수 | `missed` 상태 수 |
| 재지정 후 완료 개수 | `sourcePlanId` 있는 `done` 수 |
| 연속 달성 일수 | 일별 완료율 임계값 초과 연속일 수 |

#### 캘린더 요약 (3단계)

| 상태 | 기준 |
|------|------|
| 좋음 | 완료율 70% 이상 |
| 보통 | 완료율 30~70% |
| 회복 필요 | 완료율 30% 미만 또는 missed 비중 큼 |

#### 제외 항목 (1차 버전)

- 소셜 비교 / 친구 순위
- 복합 생산성 점수
- 장문 분석 리포트
- 과도한 차트

#### 화면 톤앤매너

- 칭찬 위주, 과장하지 않음
- "못한 것"보다 "다시 이어간 것" 강조
- 점수 게임처럼 보이지 않게

## 5. 화면 구조

### 5.1 내비게이션 구조

```
앱 루트
├── [탭] 오늘 화면 (today)
├── [탭] 동기부여 화면 (motivation)
├── [오버레이] 계획 편집 화면 (editor)
└── [오버레이] 회고 화면 (reflection)
```

**오버레이를 탭으로 두지 않는 이유:** editor와 reflection은 짧은 작업 흐름이므로 탭보다 오버레이가 더 자연스럽다.

### 5.2 화면별 명세

#### 1. 오늘 화면 (Today Screen)

앱의 기본 진입점. 실행 중심 화면.

**포함 요소:**
- 원형 24시간 플래너 + 현재 시각 포인터
- 현재 계획 강조 카드
- 오늘 계획 리스트
- 완료 현황 요약 (완료 N/전체 N)
- 시작 리마인드 배너
- 종료 5분 전 `계속 진행` 배너

**가능한 행동:**
- 계획 추가 / 수정 진입
- 완료 토글
- missed 일정의 회고 진입
- missed 일정의 다시 지정 진입
- 동기부여 화면 진입

#### 2. 계획 편집 화면 (Plan Editor Screen)

계획 추가, 수정, 다시 지정을 하나의 화면에서 처리.

**모드:**
- 새 계획 추가
- 기존 계획 수정
- missed 일정 다시 지정

**입력 요소:**
- 제목
- 시작 시각
- 종료 시각
- 색상 선택
- 저장 / 취소

**왜 하나로 묶는가:** 추가/수정/다시 지정의 유효성 검사 규칙이 동일하므로 분리하면 중복 발생

#### 3. 회고 화면 (Reflection Screen)

missed 일정에 대한 회고 메모 저장.

**포함 요소:**
- 대상 일정 요약
- 회고 입력 필드
- 저장 / 취소

**원칙:** 별도 저널 기능으로 확장하지 않음. 첫 버전은 missed 일정 맥락 안에서만 열림.

#### 4. 회복 상세 시트 (Recovery Detail Sheet)

독립 전체 화면이 아닌 오늘 화면 내 바텀 시트.

**포함 요소:**
- 회복 상태 설명 (회고 다시 보기 / 다시 지정됨 / 회복 진행 중 등)
- 후속 일정 정보
- 다시 진입 액션

#### 5. 라벨 설정 화면 (Label Settings Screen)

사용자 커스터마이징 가능한 문구 설정.

**설정 가능 항목 (7개):**
- 상태 라벨 4개: `current`, `pending`, `done`, `missed`
- 액션 라벨 3개: `completeNow`, `reflection`, `reschedule`

**설정 불가 항목 (1차 버전):**
- 오류 문구
- 관찰 패널 문구
- 섹션 제목

#### 6. 동기부여 화면 (Motivation Screen)

월간 성과 확인 및 지속 동기 신호 화면.

**레이아웃 순서:**
1. 월 헤더 + 한 줄 요약 문구
2. 핵심 성과 카드 3개 (완료 개수 / 완료율 / missed 수)
3. 연속 달성 강조 카드
4. 월간 캘린더 요약 (3단계 색상)
5. 회복 관련 보조 카드

### 5.3 주요 사용자 흐름

#### A. 메인 실행 흐름

```
오늘 화면 진입
  → 현재 계획 확인
  → (시작 리마인드) 닫기 또는 지금 완료
  → 완료 토글
```

#### B. 계획 편집 흐름

```
오늘 화면에서 추가/수정 진입
  → 계획 편집 화면에서 저장
  → 오늘 화면으로 복귀
```

#### C. missed 회복 흐름

```
일정이 missed 상태로 전환
  → 오늘 화면 카드에서 회고 또는 다시 지정 선택
  → 회고: 회고 화면 → 저장 → 복귀
  → 다시 지정: 계획 편집 화면 → 저장 → 복귀
```

#### D. 종료 5분 전 흐름

```
종료 5분 전 인앱 배너 표시
  → 사용자가 계속 진행 → 배너 닫힘
  → 종료 후 미완료 시 missed 카드 흐름
```

#### E. 동기부여 확인 흐름

```
오늘 화면 또는 요약 카드에서 동기부여 화면 진입
  → 이번 달 완료 개수 / 완료율 / missed / 연속 달성 확인
  → 필요하면 오늘 화면으로 복귀해 다음 계획 실행
```

## 6. 기술 스택

### 6.1 웹 MVP

| 항목 | 기술 |
|------|------|
| 프레임워크 | Next.js |
| 언어 | TypeScript |
| UI | React |
| 시각화 | SVG (원형 플래너) |
| 유효성 검사 | Zod |
| 날짜 유틸 | date-fns |
| 테스트 | Vitest, Testing Library, Playwright |

### 6.2 모바일 앱 (현재 진행 중)

| 항목 | 기술 |
|------|------|
| 런타임 | Expo ~55.0 + React Native 0.83 |
| 언어 | TypeScript |
| 내비게이션 | expo-router |
| 저장소 | @react-native-async-storage/async-storage |
| 알림 | expo-notifications |
| 시각화 | react-native-svg (원형 플래너) |

### 6.3 저장소 전략

| 단계 | 저장소 | 이유 |
|------|--------|------|
| 1차 | AsyncStorage | 단순 key-value, 현재 PlansStore 계약과 직접 호환 |
| 2차 (검토) | SQLite | 월간 집계, 날짜 범위 조회, 히스토리 확장 시 |

**AsyncStorage → SQLite 전환 트리거:**
- 다일자 히스토리가 기본 기능으로 필요할 때
- 통계/검색/필터가 핵심으로 올라올 때
- 서버 동기화 이전 구조화 레코드 모델이 필요할 때

## 7. 아키텍처

### 7.1 핵심 원칙

- **도메인 중심 분리**: 비즈니스 로직은 UI와 분리
- **Provider 인터페이스**: 저장소, 알림, 시간 소스는 인터페이스 뒤로 추상화
- **플랫폼 독립 도메인**: 시간 계산, 일정 규칙은 웹/앱 공용 모듈 유지
- **테스트 가능성**: 현재 시각 계산은 순수 함수로 유지

### 7.2 레이어 구조

```
types → config → repo → service → runtime → ui
```

- 상위 레이어는 바로 아래 레이어 또는 명시적 provider만 사용
- UI는 도메인 로직을 직접 가지지 않음

### 7.3 웹/앱 공용 계층 vs 앱 전용 계층

**공용 (재사용):**
- 일정 데이터 모델, 시간 슬롯 계산
- 현재 계획 판정 로직
- 완료/missed 상태 규칙
- 저장소 인터페이스 계약
- 알림 인터페이스 계약 (sync / schedule / cancel)
- 라벨 설정 키와 기본값

**앱 전용 (재구성):**
- 원형 플래너 React Native 렌더링
- 시간 입력 위젯 (네이티브)
- 바텀 시트 / 전체 화면 전환
- 로컬 알림 연결 (expo-notifications)
- 네이티브 저장소 구현

### 7.4 모바일 앱 내비게이션 계약

```
tabs/
  today       ← 반복 방문 1차 공간
  motivation  ← 반복 방문 1차 공간
editor        ← 오버레이 (짧은 작업 흐름)
```

## 8. 핵심 타입 정의

```ts
// 날짜 키
type PlanDateKey = string; // "YYYY-MM-DD"

// 계획 상태
type PlanStatus = "pending" | "done" | "missed";

// 날짜가 포함된 계획 기록
type DatedPlanRecord = DailyPlan & { date: PlanDateKey };

// 날짜별 계획 맵 (저장소 기본 단위)
type PlannerRecordMap = Record<PlanDateKey, DatedPlanRecord[]>;

// 일별 요약
type DailySummary = {
  date: PlanDateKey;
  plannedCount: number;
  completedCount: number;
  missedCount: number;
  completionRate: number;
  rescheduledCount: number;
  completedAfterRescheduleCount: number;
  reflectionCount: number;
};

// 월간 동기부여 요약
type MonthlyMotivationSummary = {
  monthKey: string;
  completedCount: number;
  completionRate: number;
  missedCount: number;
  completedAfterRescheduleCount: number;
  streakDays: number;
};

// 캘린더 날짜 상태
type CalendarDayTone = "good" | "mid" | "watch";
```

## 9. 1차 버전 범위 (MVP 범위)

### 포함

- 오늘 화면의 24시간 원형 플래너
- 계획 등록 / 수정 / 삭제
- 시작/종료 시간 입력과 시간 겹침 차단
- 현재 시각 포인터와 현재 계획 강조
- 완료 토글과 하루 완료 현황 요약
- 시작 리마인드 (OS 알림 + 인앱 배너)
- 종료 5분 전 `계속 진행` 알림
- missed 회복 흐름 (회고 / 다시 지정, 최대 3회)
- 월간 동기부여 화면 (완료율 / streak / 캘린더 요약)
- 로컬 저장소 기반 데이터 유지

### 제외

- 가족 공유 / 다중 사용자
- 외부 캘린더 연동
- 고급 통계 / 분석 리포트
- 반복 일정 자동화
- 서버 기반 동기화
- 소셜 비교 / 랭킹
