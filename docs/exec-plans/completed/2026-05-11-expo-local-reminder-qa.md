# Expo Local Reminder QA

## Goal

Expo 앱의 OS 로컬 알림이 실제 시뮬레이터에서 권한 요청, 시작 5분 전 알림, 종료 5분 전 완료 유도 알림, 일정 변경 후 재예약까지 의도대로 동작하는지 확인한다.

## Scope

- Metro/dev-client 실행 상태 확인
- 시뮬레이터에서 알림 권한 허용 여부 확인
- 짧은 재현 시나리오로 시작 알림과 종료 알림 수신 확인
- 계획 수정/삭제 후 예약 갱신 확인
- 결과 문서화

## Assumptions

- 현재 작업은 기존 브랜치 `feature/app-bootstrap-skeleton`에서 이어간다.
- working tree의 기존 변경은 보존한다.
- 실제 QA는 iOS 시뮬레이터 기준으로 진행한다.
- 재현 시간을 줄이기 위해 몇 분 뒤 시작하는 테스트 일정을 직접 만든다.

## Steps

1. 현재 Metro/dev-client와 시뮬레이터 상태를 확인한다.
2. 알림 재현용 테스트 일정을 만든다.
3. 권한 요청, 시작 5분 전 알림, 종료 5분 전 알림, 수정/삭제 후 예약 갱신을 확인한다.
4. 결과와 제한 사항을 기록한다.
5. 완료 범위와 검증 결과를 기록하고 completed로 이동한다.

## Validation

- 실제 시뮬레이터에서 알림 권한 요청 여부를 확인해야 한다.
- 시작 5분 전 알림과 종료 5분 전 알림 수신 여부를 확인해야 한다.
- 일정 변경 후 예약 갱신 또는 취소가 확인돼야 한다.
- `bash scripts/validate-docs.sh`가 통과해야 한다.

## Open Work

- 없음

## Result

- `apps/expo` Metro/dev-client와 iOS 시뮬레이터는 정상 기동 상태에서 QA를 진행했다.
- 시뮬레이터에서 테스트 일정 `알림QA`를 `15:27 - 15:35`로 생성해 시작 5분 전과 종료 5분 전 알림 재현 조건을 만들었다.
- 앱을 백그라운드로 보낸 뒤 `15:22` 시작 알림 시점을 확인했지만, 홈 화면 배너나 알림 센터 적재는 확인되지 않았다.
- 시뮬레이터에서 알림 권한 프롬프트도 별도로 노출되지 않았다.

## Verification

- 실제 iOS 시뮬레이터 수동 QA:
  - Metro/dev-client 실행 확인
  - 테스트 일정 생성 확인
  - 백그라운드 상태에서 시작 알림 시점 관찰
- `bash scripts/validate-docs.sh`

## Follow-up

- 시뮬레이터의 알림 권한 상태를 별도 확인해야 한다.
- 실제 iOS 기기 또는 권한이 초기화된 시뮬레이터에서 foreground/background 알림 수신을 다시 검증해야 한다.
- 계획 수정/삭제 후 예약 갱신 QA는 알림 수신이 확인된 뒤 이어서 진행한다.
