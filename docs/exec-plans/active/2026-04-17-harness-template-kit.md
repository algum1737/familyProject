# Harness Template Kit Plan

## Goal

현재 리포지터리 안에 다른 프로젝트로 옮겨 쓸 수 있는 하네스 시스템 구축 템플릿 세트를 만든다.

## Scope

- `하네스시스템구축방법/` 디렉터리 생성
- 재사용 가능한 메타 프롬프트 템플릿 작성
- 최소 리포지터리 뼈대 세트 작성
- 복사 및 적용 절차 문서화
- 템플릿용 문서 검증 스크립트 초안 추가

## Assumptions

- 템플릿은 특정 언어/프레임워크에 종속되지 않는 문서 중심 부트스트랩을 우선한다.
- 실제 제품 코드 대신 하네스와 기록 체계를 먼저 세우는 사용 방식을 전제로 한다.
- 사용자는 필요에 따라 제품명, 스택, 제약 조건, 검증 규칙을 후속 편집한다.

## Steps

1. 재사용 범위와 현재 리포 특화 요소 분리
2. 일반화된 하네스 엔지니어링 프롬프트 작성
3. 최소 문서/디렉터리 뼈대 세트 생성
4. 적용 가이드와 체크리스트 작성
5. 로컬 검증 수행

## Risks

- 너무 일반화하면 실제 적용성이 떨어질 수 있다.
- 템플릿 필수 항목이 부족하면 새 프로젝트에서 문서 공백이 생길 수 있다.

## Validation

- `하네스시스템구축방법/` 아래에 가이드, 템플릿, 스크립트가 모두 존재해야 한다.
- 템플릿은 현재 프로젝트 고유 제품명을 기본값으로 포함하지 않아야 한다.
- 템플릿 검증 스크립트가 최소 필수 문서 존재 여부를 검사해야 한다.
