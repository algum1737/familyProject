# App Screen Tree

## Goal

`오늘 다 했니?` 앱 전환 시 첫 구현에 필요한 화면 단위와 전환 경계를 고정한다.

이 문서는 웹 MVP의 현재 흐름을 그대로 복제하는 것이 아니라, 앱에서 어떤 기능을 독립 화면으로 두고 어떤 것은 같은 화면 안의 시트나 카드로 둘지 정하는 기준이다.

## Product Rule

- 앱 첫 버전은 `오늘 화면` 중심 단일 사용자 앱으로 시작한다.
- 핵심은 `지금 해야 할 계획 인지 -> 완료 또는 회복 행동` 흐름이다.
- 회고와 다시 지정은 별도 탭으로 키우지 않고, `missed` 이후 흐름 안에서 처리한다.
- 종료 5분 전 알림은 `계속 진행` 1액션만 유지하고, 회복 행동은 종료 직후 카드 흐름에서 처리한다.

## Primary Navigation

초기 앱은 하단 탭을 넓게 두지 않는다.

권장 구조:

1. 루트 스택
2. 기본 진입 화면은 `오늘 화면`
3. 보조 흐름은 전체 화면 또는 바텀 시트로 연다

이유:

- 현재 제품 핵심은 하루 단위 플래너 1개다.
- 통계, 가족 공유, 설정 허브 같은 추가 정보 구조가 아직 없다.
- 탭 구조를 먼저 만들면 오히려 빈 정보 구조를 고정할 가능성이 크다.

## Screen Tree

### 1. Today Screen

앱의 기본 진입점이다.

포함 요소:

- 현재 계획 카드
- 원형 24시간 플래너 또는 그 대안 표현
- 오늘 계획 리스트
- 완료 현황 요약
- 시작 리마인드 인앱 배너
- 종료 5분 전 `계속 진행` 인앱 배너

여기서 가능한 행동:

- 계획 추가 진입
- 기존 계획 수정 진입
- 완료 토글
- `missed` 일정의 회고 진입
- `missed` 일정의 다시 지정 진입

### 2. Plan Editor Screen

계획 추가와 수정, 다시 지정을 모두 처리하는 단일 편집 화면이다.

모드:

- 새 계획 추가
- 기존 계획 수정
- `missed` 일정 다시 지정

포함 요소:

- 제목 입력
- 시작 시간 입력
- 종료 시간 입력
- 색상 선택
- 저장
- 취소

같은 화면으로 묶는 이유:

- 현재 웹도 같은 상태 전이 규칙을 공유한다.
- 앱에서도 `추가/수정/다시 지정`을 따로 나누면 검증 규칙이 중복된다.

### 3. Reflection Screen

`missed` 일정의 회고 메모를 저장하는 단일 화면이다.

포함 요소:

- 대상 일정 요약
- 회고 입력
- 저장
- 취소

원칙:

- 회고는 별도 저널 화면으로 확장하지 않는다.
- 첫 버전에서는 `missed` 일정 문맥 안에서만 열린다.

### 4. Recovery Detail Sheet

별도 전체 화면이 아니라 `오늘 화면` 안에서 여는 가벼운 상세 시트다.

포함 요소:

- `회고 다시 보기`, `다시 지정 다시 보기`, `다시 지정됨`, `다시 지정 곧 시작`, `회복 진행 중` 상태 설명
- 후속 일정 정보
- 다시 진입 액션

목적:

- 리스트 카드에서 다 담기 어려운 회복 상태 설명을 보강
- 별도 탭이나 별도 섹션 없이도 회복 맥락 유지

### 5. Label Settings Screen

문구 커스터마이징은 전체 화면 또는 모달 설정 화면으로 분리한다.

포함 범위:

- 상태 라벨 4개
- 핵심 액션 라벨 3개

제외 범위:

- 오류 문구
- 관찰 패널 문구
- 섹션 제목

원칙:

- 현재 웹과 같은 7개 키만 유지한다.
- 설명성 문구는 앱 첫 버전에서 사용자 설정 범위에 넣지 않는다.

### 6. Motivation Screen

월간 동기부여와 유지 흐름을 보는 독립 화면이다.

포함 요소:

- 이번 달 완료 개수
- 이번 달 완료율
- 이번 달 `missed` 개수
- 연속 달성 카드
- 일별 달성 상태 요약
- 재지정 후 완료 / 회고 저장 같은 회복 보조 지표

원칙:

- 복잡한 분석 대시보드가 아니라 지속 동기 화면으로 둔다.
- `오늘 화면`의 핵심 실행 흐름을 방해하지 않게 분리한다.
- 데이터 기준은 `APP_MOTIVATION_DATA_MODEL.md`를 따른다.

## Flows

### A. Main Execution Flow

1. 사용자가 `오늘 화면`에 들어온다.
2. 현재 계획을 본다.
3. 필요하면 시작 리마인드에서 `지금 완료` 또는 `닫기`를 누른다.
4. 리스트 또는 플래너에서 완료 토글을 수행한다.

### B. Plan Editing Flow

1. `오늘 화면`에서 계획 추가 또는 수정 진입
2. `계획 편집 화면`에서 저장
3. 저장 후 `오늘 화면`으로 복귀

### C. Missed Recovery Flow

1. 일정이 `missed`로 바뀐다.
2. `오늘 화면` 카드에서 `회고` 또는 `다시 지정` 진입
3. 회고는 `회고 화면`, 다시 지정은 `계획 편집 화면` 재사용
4. 저장 후 `오늘 화면`으로 복귀

### D. End-5-Minute Flow

1. 종료 5분 전 `오늘 화면` 상단에 `계속 진행` 배너 노출
2. 사용자가 `계속 진행`을 누르면 배너만 닫힘
3. 종료 직후 `missed` 카드 흐름으로 전환

### E. Motivation Review Flow

1. 사용자가 `오늘 화면` 또는 요약 카드에서 월간 동기부여 화면으로 진입
2. 이번 달 완료 개수, 완료율, `missed`, 연속 달성 일수를 확인
3. 필요하면 다시 `오늘 화면`으로 돌아와 다음 계획 실행을 이어감

## Screen Boundary Rules

- `오늘 화면`은 실행 중심 화면이다.
- 편집은 `계획 편집 화면` 하나로 모은다.
- 회고는 별도 화면으로 분리하되, 탐색 허브로 키우지 않는다.
- 회복 상세는 독립 화면이 아니라 시트로 처리한다.
- 설정은 핵심 실행 흐름을 방해하지 않도록 분리한다.

## What Stays Shared With Web

- 계획 데이터 모델
- 시간 계산 로직
- 현재 계획 판정
- 완료와 `missed` 상태 규칙
- 다시 지정 최대 3회 규칙
- 종료 5분 전 `계속 진행` 정책
- 라벨 설정 키 범위

## What Becomes App-Specific

- 원형 플래너 렌더링 방식
- 시간 입력 위젯
- 바텀 시트 또는 전체 화면 전환 방식
- 로컬 알림 연결
- 네이티브 저장소 구현
