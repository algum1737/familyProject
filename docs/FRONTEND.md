# Frontend

## Current Assumption

- 제품은 사용자가 원형 24시간 계획판에서 하루의 할 일과 현재 해야 할 계획을 직관적으로 확인하는 인터랙티브 UI를 가진다.

## Frontend Rules

- UI는 제품 핵심 여정을 먼저 표현한다.
- 첫 화면은 원형 24시간 계획판을 중심으로 구성한다.
- 현재 시각 포인터와 현재 계획 강조는 MVP 핵심 인터랙션이다.
- 현재 해야 할 계획의 부채꼴은 후속 단계에서 다른 계획보다 더 진한 색, 더 높은 불투명도, 또는 별도 외곽 강조로 차별화한다.
- 상태와 비즈니스 규칙은 UI에서 분리한다.
- 검증 가능한 셀렉터와 테스트 포인트를 남긴다.
- 에이전트가 DOM, 스크린샷, 사용자 흐름을 읽고 검증할 수 있어야 한다.
