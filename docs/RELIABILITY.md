# Reliability

## Goals

- 핵심 사용자 여정이 재현 가능해야 한다.
- 실패는 재시도 가능하고 관측 가능해야 한다.
- 로컬 개발 환경에서도 최소한의 검증 루프가 있어야 한다.

## Initial Requirements

- 앱 실행/중지 방법이 문서화되어야 한다.
- 핵심 흐름에 대한 smoke test가 있어야 한다.
- 오류는 구조화 로그로 남아야 한다.


## Debugging Protocol

버그, 테스트 실패, 빌드 실패, 예기치 않은 동작은 수정부터 하지 않고 원인 조사부터 시작한다.

1. 오류 메시지, 스택 트레이스, 실패 명령을 끝까지 읽는다.
2. 재현 절차와 입력 조건을 기록한다.
3. 최근 변경, 설정, 의존성, 환경 차이를 확인한다.
4. 정상 동작 예시와 실패 동작을 비교한다.
5. 하나의 가설을 세우고 가장 작은 실험으로 확인한다.
6. 원인이 확인되면 실패를 재현하는 테스트 또는 검증 스크립트를 먼저 만든다.
7. 한 번에 하나의 수정만 적용하고 같은 검증을 다시 실행한다.
8. 같은 문제에 대해 여러 수정이 연속 실패하면 구조적 문제 가능성을 기록하고 사람에게 에스컬레이션한다.

## Evidence To Keep

- 실패 명령과 출력 요약
- 재현 단계
- 조사한 가설과 배제 근거
- 최종 원인
- 수정 범위
- 재검증 명령과 결과
