# Quality Score

## Scoring Areas

- 제품 명세 명확성
- 아키텍처 일관성
- 테스트 커버리지
- 관측 가능성
- 문서 최신성
- 에이전트 작업 가능성
- 테스트 우선 구현 준수 수준
- 작업 단위 리뷰와 완료 전 검증 수준

## Initial Rubric

- `0`: 부재
- `1`: 초안만 존재
- `2`: 문서화됨
- `3`: 검증 가능
- `4`: 자동 강제됨

## Current Baseline

- 제품 명세 명확성: 2
- 아키텍처 일관성: 1
- 테스트 커버리지: 0
- 관측 가능성: 0
- 문서 최신성: 2
- 에이전트 작업 가능성: 1

## Superpowers-Aligned Checks

- 기능 추가와 버그 수정은 가능한 한 실패하는 테스트를 먼저 남긴다.
- 테스트가 즉시 통과했다면 기존 동작을 검증한 것인지, 새 요구사항을 검증한 것인지 확인한다.
- task 완료 전에는 계획 요구사항 충족 여부와 코드 품질을 별도로 검토한다.
- 완료 선언 전에는 실제 실행한 검증 명령과 결과를 `Validation Result`에 남긴다.
- 반복되는 리뷰 지적은 문서 규칙, lint, test, CI 후보로 승격한다.
