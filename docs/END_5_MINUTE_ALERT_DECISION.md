# End-5-Minute Alert Decision

## Decision

- 앱 전환 초기안에서는 종료 5분 전 회복 알림을 `계속 진행` 1액션 구성으로 유지한다.
- `회고 열기`, `다시 지정`은 종료 5분 전 알림에 넣지 않는다.
- 회고와 다시 지정은 종료 직후 `missed` 카드 흐름에서 처리한다.

## Why

- 시작 리마인드가 이미 한 번 노출된 체인에서는 종료 5분 전 알림이 없어도 현재 계획 맥락이 유지됐다.
- 종료 5분 전 실험 배너를 `계속 진행` 1액션으로 줄였을 때 실제 화면 부담이 낮았다.
- 같은 날 관찰 표본 3건에서 `종료 전 확인 -> 계속 진행 -> 종료 직후 missed -> 회고 다시 보기`가 반복됐다.
- 종료 직후 카드의 `회고 다시 보기`와 `다시 지정` 흐름은 이미 이해 가능했고, 종료 직전까지 세 가지 액션을 싣는 것은 과하다는 초기 관찰과도 일치했다.

## Evidence

- 시작 리마인드 누적: `표시 7회 / 닫기 2회 / 완료 2회`
- 종료 직전 실험 배너 반복:
  - `관찰 표본 1`: `계속 진행 -> 종료 직후 missed -> 회고 다시 보기`
  - `관찰 표본 2`: `계속 진행 -> 종료 직후 missed -> 회고 다시 보기`
  - `관찰 표본 2` 반복 표본: `계속 진행 -> 종료 직후 missed -> 회고 다시 보기`
- 상세 관찰 로그: [RECOVERY_OBSERVATION_LOGBOOK.md](./RECOVERY_OBSERVATION_LOGBOOK.md)

## Product Rule

- 종료 5분 전에는 `pending` 일정 1건에 대해 인앱 배너를 1회만 노출한다.
- 배너 문구는 가벼운 확인 성격으로 유지한다.
- 액션은 `계속 진행` 하나만 둔다.
- 배너를 닫아도 일정 상태는 바꾸지 않는다.
- 종료 시각이 지나면 기존 규칙대로 `missed`로 전환하고, 카드에서 `회고 다시 보기`와 `다시 지정`을 보여준다.

## Revisit Triggers

- 실제 운영 표본에서 종료 5분 전 배너가 반복적으로 무시되거나 거슬린다는 반응이 2회 이상 쌓일 때
- 실제 운영 표본에서 종료 직후 회고 진입이 자주 놓쳐져 종료 전 보조 액션이 필요하다는 근거가 생길 때
- 시작 리마인드와 종료 직전 배너가 합쳐져 과한 빈도로 느껴진다는 반응이 반복될 때
- 실제로는 일을 했는데 완료 처리만 놓쳐 `missed`가 되는 사례가 반복돼, 종료 직전 완료 확인 신호의 필요성이 분명해질 때
